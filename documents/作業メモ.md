## Jquants 公式 MCP サーバー導入

- [Jquants MCP - GitHub](https://github.com/J-Quants/j-quants-doc-mcp)
- [Github 公式 MCP 導入手順](https://docs.github.com/ja/copilot/how-tos/provide-context/use-mcp/extend-copilot-chat-with-mcp)

1. uv はすでにホストにインストール済みだった
2. `uv tool install git+https://github.com/J-Quants/j-quants-doc-mcp.git`
3. `mkdir .vscode`
4. `touch .vscode/mcp.json`
5. ファイルに以下を記述
   ```json
   {
     "servers": {
       "j-quants-doc-mcp": {
         "command": "uvx",
         "args": ["j-quants-doc-mcp"]
       }
     }
   }
   ```
6. Agent モードの GitHub Copilot に対し、「j-quants-doc-mcp を使えるかテストして」と質問
7. ヘルスが正常であり、正常な返答が返ってくることを確認

## 本番インフラの Terraform + GCP 移行

1. Google Cloud 関係
   1. 仕事用 Google アカウントで GCP にログイン
   2. Google Cloud コンソールでプロジェクト作成
   3. 課金有効化は過去使ったことがあるためかすでに行われていた
   4. 予算アラート設定 5000 円/月
   5. Google Cloud CLI インストール ([公式](https://docs.cloud.google.com/sdk/docs/install-sdk?hl=ja))
   6. `gcloud init` で初期化、プロジェクト選択
   7. `gcloud auth application-default login` で認証 (~/.config/gcloud/application_default_credentials.json の作成)
   8. `gcloud config set project investsupporter` でプロジェクト設定 (コマンドごとに --project オプションを入力する手間を省く)
   9. tfstate 保存用 GCS バケット作成、バージョニング有効化、パブリックアクセス不可設定
2. Terraform 関係
   1. tenv のインストール ([公式 README](https://github.com/tofuutils/tenv) に従う)
   2. `tenv terraform install` で Terraform インストール (バージョン 1.14.3)
   3. tfvars の allowed_ssh_ips を IAP 用 IP 範囲 (35.235.240.0/20) のみに修正
   4. tfvars の oslogin_members, osadminlogin_members に自分の Google アカウントを追加
   5. `gcloud compute images list` で使用可能イメージを確認、使用イメージを Debian 13 に変更
   6. `terraform init` で初期化
   7. `terraform fmt` でコードフォーマット
   8. `terraform plan -out="plan.tfplan"` で差分確認
   9. `terraform apply "plan.tfplan"` で適用
3. 確認
   1. tfstate 保存用 GCS バケットに tfstate ファイルが保存されていることを確認
   2. Compute Engine インスタンスにブラウザコンソールから ssh 接続できること、`docker compose version` が成功することを確認
   3. `gcloud compute ssh invest-supporter-app --zone asia-northeast1-a --tunnel-through-iap` でローカルから ssh 接続できることを確認
   4. 一応目標レベルの Google Cloud インフラが構築できた

## アプリケーションの初期設定と CICD 設定
1. 初回起動
   1. `gcloud compute ssh invest-supporter-app --zone asia-northeast1-a --tunnel-through-iap`
   2. `git clone https://github.com/root-5/InvestSupporter.git`
   3. `exit` で一旦ログアウト
2. 秘匿データのコピー、復元
   1. `bash cicd/sync_hidden_files.sh PROJECT_ID="your-gcp-project-id"` で秘密情報とバックアップ (.env, .dump) をコピー
   2. `gcloud compute ssh invest-supporter-app --zone asia-northeast1-a --tunnel-through-iap`
   3. `docker compose up db -d` で DB コンテナを起動
   4. `docker compose exec db bash /var/lib/postgresql/backup/restore.sh` で DB を復元
3. Workload Identity の設定
   1. Terraform による Workload Identity Federation 関連インフラの導入
4. GitHub Actions の設定
   1. `.github/workflows/deploy.yaml` の作成
   2. GitHub Actions から gcloud CLI を使用して実行されるデプロイスクリプト `cicd/update_app.sh` の作成
5. GitHub 上の設定
   1. 環境変数設定 (Settings > Secrets and variables > Actions > New repository secret)
      - `GCP_PROJECT_ID` : GCP プロジェクト ID
      - `GCP_WORKLOAD_IDENTITY_PROVIDER` : Workload Identity プロバイダー、`gcloud iam workload-identity-pools providers describe ...` コマンドで確認可能
      - `GCP_ZONE` : GCP ゾーン
6. 動作確認
   1. main ブランチにコミットし、GitHub Actions が成功するか確認
