## 修正手順

### 本番インフラの Terraform + GCP 移行

#### 1. 移行方針

- **プラットフォーム**: Google Cloud Platform (GCP)
- **構成**: Google Compute Engine (GCE) インスタンス 1 台で Docker Compose を実行する構成とする（リフト＆シフト）。
- **OS**: Ubuntu 22.04 LTS (Docker, Docker Compose が利用しやすい環境)
- **IaC ツール**: Terraform
- **バージョン管理**: `tenv` を使用

#### 2. Terraform 管理リソース

- **VPC Network**: デフォルトではなく専用の VPC を作成
- **Subnet**: 東京リージョン (asia-northeast1)
- **Firewall**:
  - SSH (22): 管理者 IP のみ許可 (運用時設定)
  - App (7203 -> 8080): 必要に応じて公開、または IAP 経由
  - DB (5432): 外部公開しない (内部通信のみ)
- **Compute Engine**:
  - Machine Type: `e2-medium` (Go アプリ + DB を動かすため micro では厳しい可能性あり)
  - Disk: 標準永続ディスク
- **Static IP**: 固定 IP アドレスの確保

#### 3. ディレクトリ構成

`infra/terraform/` 配下に Terraform コードを配置する。

#### 4. 作業ログ

- [x] Terraform ディレクトリ作成

- [x] Terraform コード作成 (main.tf, variables.tf, etc.)

#### 5. 構築・デプロイ手順 (ユーザー実施)

1. **事前準備**

   - GCP プロジェクトを作成し、課金を有効化する。
   - 以下の API を有効化する。
     - Compute Engine API
   - `gcloud` CLI をインストールし、認証を行う。
     ```bash
     gcloud auth application-default login
     gcloud config set project [PROJECT_ID]
     ```
   - `tenv` をインストールする。

2. **Terraform 実行**

   - `infra/terraform/terraform.tfvars.example` をコピーして `infra/terraform/terraform.tfvars` を作成し、`project_id` などを設定する。
   - 初期化と適用:
     ```bash
     cd infra/terraform
     tenv tf init
     tenv tf apply
     ```

3. **アプリケーションデプロイ**
   - 作成されたインスタンスへ SSH 接続する (OS Login が有効になっているため `gcloud` コマンド推奨)。
     ```bash
     gcloud compute ssh [INSTANCE_NAME] --zone [ZONE]
     ```
   - リポジトリをクローンするか、ファイルを転送する。
   - `.env` ファイル等を設定する。
   - Docker Compose で起動する。
     ```bash
     docker compose -f compose.yaml up -d --build
     ```
